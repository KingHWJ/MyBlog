# MyBlog 技术方案

## 1. 站点定位

MyBlog 是一个面向后端与大数据学习者的中文知识库技术站，首版强调：

- 知识入口清晰
- 专题结构稳定
- 内容可持续扩展
- 发布与维护成本可控

## 2. 内容结构

站点采用“专题文档 + 更新文章”的双轨结构：

- `roadmap/`：学习路线与方法论
- `backend/`：后端基础主线
- `bigdata/`：大数据专题
- `updates/`：更新归档与独立文章

其中：

- 专题文档用于长期维护和持续修订
- 更新文章用于记录新的输出，再按需要沉淀回专题

## 3. Frontmatter 规范

所有内容页统一使用以下 frontmatter 字段：

- `title`：页面标题
- `description`：页面简介
- `date`：发布日期，格式建议为 `YYYY-MM-DD`
- `tags`：标签数组
- `category`：内容分类
- `featured`：是否出现在首页推荐
- `draft`：是否为草稿

## 4. 导航与侧边栏

顶部导航固定为：

- 首页
- 学习路线
- 后端基础
- 大数据
- 归档/更新

侧边栏采用手工配置，确保目录层次稳定，首版不引入自动生成规则。

## 5. 工程与发布

- 包管理器：`pnpm`
- 站点框架：`VitePress`
- 测试：`Vitest`
- 部署：`GitHub Pages`

发布流程：

1. 在开发分支上完成修改
2. 本地运行测试与构建验证
3. 合并时使用 `--no-ff`
4. 由 GitHub Actions 负责构建并发布到 GitHub Pages

## 6. 开发约束

- 不在主分支直接开发
- Git 提交按小模块拆分，并使用中文提交信息
- 代码和配置中的注释优先使用中文
- 开发过程中同步维护技术文档，避免实现与文档脱节
