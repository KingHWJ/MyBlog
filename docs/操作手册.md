# 操作手册

本文档用于记录 MyBlog 的常见维护动作，目标是让日常增删改查和启动发布都有固定入口。

## 1. 开发前准备

### 1.1 Node 与依赖

推荐使用 Node.js 22 LTS：

```bash
cat .nvmrc
pnpm install
```

### 1.2 创建开发分支

不要直接在 `main` 上开发：

```bash
git switch main
git pull --ff-only
git switch -c codex/your-task
```

### 1.3 创建 worktree

仓库的 worktree 统一放在 `~/Codes/worktree` 下，并对照原项目在 `~/Codes` 下的层级结构。

先计算目录：

```bash
pnpm worktree:path codex/your-task
```

当前项目会输出类似：

```text
/Users/hwj/Codes/worktree/MyProjects/MyBlog/codex-your-task
```

再创建 worktree：

```bash
BRANCH="codex/your-task"
TARGET="$(pnpm --silent worktree:path "$BRANCH")"
git worktree add "$TARGET" -b "$BRANCH" main
```

创建完成后进入新目录并安装依赖：

```bash
cd "$TARGET"
pnpm install
```

如果旧目录 `~/Codes/worktree/MyProjects/MyBlog` 已经被直接当成某个 worktree 使用，需要先迁移旧 worktree，再切换到新的“项目目录 / 分支子目录”结构。

## 2. 常用启动与维护命令

### 2.1 启动开发环境

```bash
pnpm docs:dev
```

### 2.2 预览构建结果

```bash
pnpm docs:build
pnpm docs:preview
```

### 2.3 运行测试

```bash
pnpm test
```

### 2.4 发布前检查

```bash
pnpm test
pnpm docs:build
git status --short --branch
```

### 2.5 合并回主分支

```bash
git switch main
git merge --no-ff codex/your-task
```

## 3. 文档的增删改查

### 3.1 新增文档

先确认内容属于哪个目录：

- 学习路线：`docs/roadmap/`
- 后端专题：`docs/backend/`
- 大数据专题：`docs/bigdata/`
- 更新文章：`docs/updates/`

新增文档时，先复制现有页面的 frontmatter 结构，至少维护：

```yaml
title:
description:
date:
tags:
category:
featured:
draft:
```

新增后别忘了同步更新侧边栏或入口链接。

### 3.2 修改文档

直接编辑对应 Markdown 文件即可。常见入口：

- 首页：`docs/index.md`
- 学习路线：`docs/roadmap/`
- 后端专题：`docs/backend/`
- 大数据专题：`docs/bigdata/`
- 更新归档：`docs/updates/`

修改后建议执行：

```bash
pnpm docs:dev
```

### 3.3 删除文档

删除 Markdown 文件后，必须同步清理以下引用：

- `docs/.vitepress/config.ts` 中的侧边栏链接
- 首页或专题页中的手工链接
- 更新归档页中的列表入口

如果只删文件不删入口，站点会出现死链。

### 3.4 查询文档

建议使用 `rg` 搜索标题、标签或路径：

```bash
rg "MySQL 索引设计" docs
rg "featured: true" docs
rg "/backend/network/" docs
```

## 4. 网站模块头的增删改查

这里的“模块头”主要指顶部导航、首页 Hero、首页专题卡片、侧边栏分组和最近更新模块。

### 4.1 顶部导航

文件：

- `docs/.vitepress/config.ts`

位置：

- `themeConfig.nav`

可执行操作：

- 新增导航项：在 `nav` 数组中新增对象
- 修改导航项：调整 `text` 和 `link`
- 删除导航项：移除对应对象
- 查询导航项：直接搜索 `themeConfig.nav`

### 4.2 侧边栏分组

文件：

- `docs/.vitepress/config.ts`

位置：

- `themeConfig.sidebar`

可执行操作：

- 新增页面后，补对应分组条目
- 调整顺序时，直接调整数组顺序
- 删除页面后，移除对应 `link`

### 4.3 首页 Hero

文件：

- `docs/index.md`

位置：

- 顶部 frontmatter 的 `hero`

可执行操作：

- 修改站点标题、副标题、按钮文案和链接
- 新增/删除按钮

### 4.4 首页专题卡片

文件：

- `docs/.vitepress/theme/components/HomeTopicGrid.vue`

可执行操作：

- 新增卡片：在 `topics` 数组中新增对象
- 修改卡片：调整标题、说明、链接和徽标
- 删除卡片：移除对象

### 4.5 最近更新模块

文件：

- `docs/.vitepress/theme/components/RecentUpdates.vue`
- `docs/.vitepress/theme/data/updates.data.ts`
- `docs/.vitepress/theme/data/content-utils.ts`

可执行操作：

- 新增更新文章：在 `docs/updates/` 下新增文档，首页会自动读取
- 修改排序规则：调整 `buildRecentUpdates`
- 修改展示卡片样式：调整 `RecentUpdates.vue` 与 `custom.css`

### 4.6 站点样式、Logo 与图标

文件：

- 样式：`docs/.vitepress/theme/custom.css`
- Logo：`docs/public/logo-mark.svg`
- Favicon：`docs/public/favicon.svg`

## 5. 常见功能操作

### 5.1 新增一个专题

1. 在对应目录下创建专题首页和内容页
2. 在 `docs/.vitepress/config.ts` 中补侧边栏
3. 如需首页展示，在 `HomeTopicGrid.vue` 中新增卡片
4. 运行 `pnpm docs:build`

### 5.2 新增一篇更新文章

1. 在 `docs/updates/` 下创建新 Markdown
2. 补全 frontmatter
3. 确认 `draft: false`
4. 首页最近更新会自动读取，无需手工维护列表

### 5.3 更换 Logo 或站点图标

1. 替换 `docs/public/logo-mark.svg` 或 `docs/public/favicon.svg`
2. 如有尺寸或路径变化，检查 `docs/.vitepress/config.ts`
3. 运行 `pnpm docs:build` 验证资源引用

### 5.4 调整 GitHub Pages 发布流程

文件：

- `.github/workflows/deploy.yml`

常见动作：

- 修改 Node 版本
- 增加构建前检查
- 调整触发分支

## 6. 提交与合并建议

- 小步提交，一个小功能一个提交
- 提交信息使用中文
- 合并到主分支时统一使用 `--no-ff`
- 修改代码时，同步更新中文技术文档

## 7. 推荐的维护顺序

1. 建分支或 worktree
2. 修改内容或模块配置
3. 本地启动检查页面效果
4. 运行测试和构建
5. 小步中文提交
6. 用 `--no-ff` 合并到 `main`
